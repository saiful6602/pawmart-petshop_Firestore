# PawMart — Pet Shop E-Commerce App (Flutter + Cloud Firestore)

A formal, industry-level pet shop shopping app built with Flutter and Cloud Firestore.
Customers browse a real-time product catalogue, view product details, and place orders —
all synced live through Firebase.

Built with a clean, feature-based architecture (`core/`, `data/`, `features/`, `shared/`),
the same structure used in professional Flutter codebases.

## Screens

- **Home Screen** — real-time product catalogue from Firestore, category filter, hero banner.
- **Product Details Screen** — full product info, specs, price, Buy Now button.
- **Checkout Screen** — delivery form (name, phone, address, quantity) → places an order in Firestore.
- **My Orders Screen** — the signed-in user's own order history (read-only), reachable from the receipt icon in the app bar.

## Why no product photos?

Real product photography is licensed content, so this app avoids bundling stock/product
images. Instead, each product gets a clean icon + accent color derived from its category
(a common, professional pattern for demo/catalogue apps). You can swap in real photos later
by extending `Product` with an `imageUrl` field and using `Image.network(...)` in
`shared/widgets/product_image.dart`.

## How the data flows (Firebase)

| Collection | Who writes | Who reads |
|---|---|---|
| `products` | Only you, manually, from the Firebase Console | Everyone (public catalogue) |
| `orders` | The signed-in customer, when they check out | Only that same customer, for their own orders |

Every customer is signed in **anonymously** the moment the app opens (no login screen) —
this gives each device a private ID so their order history stays theirs alone, while the
product catalogue stays open to everyone.

## Step 1 — Install prerequisites

- Install Flutter SDK: https://docs.flutter.dev/get-started/install
- Install VS Code: https://code.visualstudio.com/
- In VS Code, install the **Flutter** extension (Dart comes with it).
- Verify your setup:
  ```
  flutter doctor
  ```
  Fix anything marked `[✗]` before continuing.

## Step 2 — Open the project

1. Unzip/copy the `pet_shop_app` folder anywhere on your computer.
2. VS Code → File → Open Folder → select `pet_shop_app`.
3. Open a terminal (`` Ctrl+` ``) and run:
   ```
   flutter create .
   ```
   (this generates the missing `android/`, `ios/`, `web/` platform folders — this repo only ships the Dart source)
4. Then:
   ```
   flutter pub get
   ```

## Step 3 — Create a NEW Firebase project

1. Go to https://console.firebase.google.com/ → **Add project**.
2. Name it something like `pawmart-petshop` and finish the wizard.
3. Left menu → **Build → Firestore Database → Create database**.
4. Choose **Start in test mode** for now (we'll lock it down with real rules in Step 6).
5. Pick any region close to you.

## Step 4 — Connect your Flutter app to Firebase

From the `pet_shop_app` project root:

```bash
# 1. Install the Firebase CLI (if you don't have it)
npm install -g firebase-tools

# 2. Log in to your Google/Firebase account
firebase login

# 3. Install the FlutterFire CLI
dart pub global activate flutterfire_cli

# 4. Configure — this auto-generates a REAL lib/firebase_options.dart
flutterfire configure
```

When prompted:
- Select the **new** Firebase project you just created in Step 3.
- Select platforms: **web** and **android** (add `ios`/`macos`/`windows` too if you need them).
- If asked to override the existing `firebase_options.dart`, say **yes**.

## Step 5 — Enable Anonymous Authentication

1. Firebase Console → your project → **Build → Authentication → Get started**.
2. **Sign-in method** tab → click **Anonymous** → **Enable** → **Save**.

This lets the app sign every customer in silently, with no login form.

## Step 6 — Publish the Firestore security rules

1. Firebase Console → **Firestore Database → Rules** tab.
2. Delete everything in the editor and paste the contents of `firestore.rules` from this repo:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /products/{productId} {
         allow read: if true;
         allow write: if false;
       }
       match /orders/{orderId} {
         allow create: if request.auth != null
                       && request.resource.data.userId == request.auth.uid;
         allow read: if request.auth != null
                     && resource.data.userId == request.auth.uid;
         allow update, delete: if false;
       }
     }
   }
   ```
3. Click **Publish**.

## Step 7 — Add sample products (manually, via Console)

Since products are managed by you (not the app), add a few to get started:

1. Firebase Console → **Firestore Database → Data** tab.
2. **Start collection** → Collection ID: `products` → **Next**.
3. For the first document, leave the Document ID as **Auto-ID**, and add these fields:

   | Field | Type | Example value |
   |---|---|---|
   | `name` | string | `Premium Dog Dry Food 3kg` |
   | `description` | string | `Balanced nutrition with real chicken, vegetables, and omega-3 for a shiny coat and strong immunity. Suitable for all breeds.` |
   | `category` | string | `Dog Food` (must exactly match one of: `Dog Food`, `Cat Food`, `Toys`, `Accessories`, `Grooming`, `Health & Wellness`, `Bedding`, `Aquarium`) |
   | `price` | number | `1450` |
   | `rating` | number | `4.6` |
   | `reviewCount` | number | `84` |
   | `stock` | number | `25` |
   | `createdAt` | timestamp | click the clock icon → "current date" |

4. Click **Save**.
5. Repeat **Add document** a few more times with different categories (e.g. a cat food, a chew toy, a leash, a grooming brush, a fish tank filter) so the Home screen's category filter has something to show. 6–10 products makes a good demo catalogue.

## Step 8 — Run the app

```bash
flutter pub get
flutter run -d chrome
```
Or press **F5** in VS Code with a device selected.

Test the flow:
1. Home screen loads your products live from Firestore.
2. Tap a product → see full details.
3. Tap **Buy Now** → fill in the delivery form → **Place Order**.
4. Open Firebase Console → Firestore → `orders` collection — your order should appear instantly.
5. Tap the receipt icon (top-right of Home) → **My Orders** → see your own order history.

## Notes for grading / reviewers

- **Create**: `PurchaseScreen` → `OrderRepository.placeOrder()` writes a new order document.
- **Read**: `HomeScreen` → `ProductRepository.getProducts()` streams the catalogue in real time; `MyOrdersScreen` → `OrderRepository.getMyOrders()` streams the signed-in user's own orders.
- **Security**: Firestore rules (`firestore.rules`) restrict `products` writes to the Console only, and scope every order read/write to the order's own `userId`, matched against Firebase Anonymous Auth's `request.auth.uid`.
- Firestore collections used: `products` (fields: `name`, `description`, `category`, `price`, `rating`, `reviewCount`, `stock`, `createdAt`) and `orders` (fields: `userId`, `productId`, `productName`, `unitPrice`, `quantity`, `totalAmount`, `customerName`, `phone`, `address`, `createdAt`).
